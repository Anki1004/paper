Q1 Von Neumann architecture & functional blocks of a computer

⚡ Core Intuition

One memory holds both instructions and data; CPU fetches them through the same bus, one at a time.

📖 Concept Explained
  • Stored-program concept — instructions live in memory, not hard-wired.
  • Five functional blocks: CPU, Memory, Input, Output, Bus.
  • CPU = ALU + Control Unit + Registers.
  • Single bus = "Von Neumann bottleneck" (one fetch at a time).
  • Harvard architecture splits instruction and data memory — faster.
🔗 Connects to stored program bus system fetch–execute RTL
📐 Exam Answer Structure
  • 1-line definition (stored-program).
  • Block diagram with 5 blocks + 3 buses.
  • Function of each block — bullets, not prose.
  • Mention Von Neumann bottleneck → contrast with Harvard.
💡 Example

Loading x = a + b: CPU fetches instruction from memory, then fetches a, then b — three separate memory accesses on one bus.

🧠 Mnemonic — "CMIOB" · CPU · Memory · Input · Output · Bus.
✅ Model Exam Answer
  1. Def: Computer where instructions and data share one memory (stored-program, J. von Neumann, 1945).
  2. Diagram with CPU(ALU+CU+Reg), Memory, I/O, Bus.
  3. Functions: ALU computes; CU decodes & sequences; Memory stores; I/O communicates; Bus carries addr/data/control.
  4. Limitation: bottleneck — only one fetch at a time.
  5. Significance: foundation of every general-purpose CPU since 1950.

Q2 Instruction execution cycle (Fetch–Decode–Execute) with RTL

⚡ Core Intuition

CPU runs an endless loop: "Get the next instruction. Figure out what it means. Do it." Each phase is just register-to-register copies driven by the clock.

📖 Concept Explained
  • PC — Program Counter, holds address of next instruction.
  • IR — Instruction Register, holds current instruction.
  • MAR — Memory Address Register (drives address bus).
  • MDR — Memory Data Register (buffers data bus).
  • Cycle = Fetch → Decode → (Operand fetch) → Execute → Write-back.
🔗 Connects to PC, IR, MAR, MDR RTL micro-ops Control Unit Pipelining
📐 Exam Answer Structure
  • Block diagram of CPU registers + memory.
  • Phase-by-phase narrative (Fetch/Decode/Execute).
  • RTL micro-operations table for the fetch phase.
  • One worked example (e.g. ADD R1, R2).
RTL — Fetch Cycle (3 clock pulses)

T1: MAR ← PC  |  T2: MDR ← M[MAR], PC ← PC+1  |  T3: IR ← MDR

Example: ADD R1, R2  (R1 ← R1 + R2)

T1: MAR ← PC
T2: MDR ← M[MAR],  PC ← PC + 1
T3: IR  ← MDR              ; instruction now in IR
T4: decode opcode (CU)     ; identifies ADD
T5: R1 ← R1 + R2 (ALU)     ; execute
done — return to T1 for next instruction
💡 Example

If PC = 200 holds ADD R1,R2, fetch puts opcode in IR; CU asserts ALU's ADD line; result lands in R1 within one cycle.

🧠 Mnemonic — "FDE Loop" · Fetch · Decode · Execute · (repeat). The CPU never stops.
✅ Model Exam Answer
  1. Def: Instruction cycle = sequence of steps CPU performs to process one instruction.
  2. Diagram showing PC, MAR, MDR, IR, CU, ALU.
  3. Phases — Fetch (bring instr from memory), Decode (CU interprets opcode), Execute (ALU acts), then PC ← PC+1.
  4. RTL table for fetch (T1–T3).
  5. Worked example with ADD R1,R2.
  6. Significance: every program = millions of these cycles.

Q3 Addressing modes — syntax, working, examples, comparison

⚡ Core Intuition

An instruction's address field is a recipe for finding the operand, not always the operand itself. Different recipes = different addressing modes.

📖 Concept Explained
  • Immediate — operand IS the address field. Fastest.
  • Direct — address field = memory location of operand.
  • Indirect — address field points to a pointer in memory.
  • Register — operand is in a CPU register.
  • Register indirect — register holds the memory address.
  • Indexed — EA = base address + index register (arrays).
  • Relative — EA = PC + offset (branches).

Q4 Signed number representations — sign-magnitude, 1's complement, 2's complement

⚡ Core Intuition

How do we store a minus sign in pure binary? Three tricks: dedicate a sign bit (sign-magnitude), flip all bits (1's complement), or flip-and-add-1 (2's complement). Only 2's complement makes subtraction = addition.

📖 Concept Explained
  • Sign-magnitude — MSB = sign (0=+, 1=−); rest = magnitude. Two zeros (+0, −0). Range = ±(2ⁿ⁻¹−1).
  • 1's complement — negative = flip all bits. Two zeros. End-around-carry on add.
  • 2's complement — negative = flip + add 1. Single zero. Range = −2ⁿ⁻¹ to +2ⁿ⁻¹−1. Used everywhere.
🔗 Connects to binary arithmetic adder/subtractor overflow detection
📐 Exam Answer Structure
  • Define each scheme in 1 line.
  • Worked table: +5 and −5 in 8-bit, all three schemes.
  • State ranges and zero count.
  • End with "why 2's complement wins": subtraction = addition.
Worked: A − B  using 2's complement (4-bit)
  A = 6 = 0110
  B = 4 = 0100
  −B = 1's-comp(0100) + 1 = 1011 + 1 = 1100
  A + (−B) = 0110 + 1100 = 1 0010
              discard carry → 0010 = 2 ✓
💡 Example

Compute 7 − 3 in 4-bit 2's-comp: 0111 + 1101 = 1 0100 → discard carry → 0100 = 4. ✓

🧠 Mnemonic — "FA1" · Flip · Add 1. The whole 2's-complement trick in three letters.
✅ Model Exam Answer
  1. Def: Signed number = binary scheme that encodes sign + magnitude.
  2. Three schemes: sign-magnitude, 1's, 2's complement.
  3. Table with +5/−5 in 8-bit.
  4. Numerical: subtract using 2's complement.
  5. Pros/cons: 2's complement → single zero, sub = add, one ALU.

Q5 IEEE 754 floating-point representation — convert 12.375 (single precision)

⚡ Core Intuition

Floating point = scientific notation in binary. Three fields: sign, biased exponent, mantissa. The bias lets us compare floats with integer comparison.

📖 Concept Explained
  • Single precision — 32 bits = 1 sign + 8 exp + 23 mantissa. Bias = 127.
  • Double precision — 64 bits = 1 sign + 11 exp + 52 mantissa. Bias = 1023.
  • Normalized form: 1.xxxx × 2^E. The leading 1 is hidden.
  • Stored exponent = true exponent + bias.
  • Special values: 0 (all zeros), ∞ (exp=all-1, mant=0), NaN (exp=all-1, mant≠0).
🔗 Connects to scientific notation bias precision loss FP arithmetic
📐 Exam Answer Structure
  • Draw the 32-bit layout (1+8+23).
  • Step 1 → convert |n| to binary.
  • Step 2 → normalize to 1.xxx × 2ᴱ.
  • Step 3 → biased exp = E + 127.
  • Step 4 → fill mantissa (drop leading 1).
  • Assemble final 32-bit string.
Bonus PYQ trace: −262.125 → IEEE 754 single (LOC_CSE24 Q2b)
  |n|        = 262.125
  binary     = 100000110.001
  normalize  = 1.00000110001 × 2⁸

  sign       = 1  (negative)

Always store E + 127 (single) or E + 1023 (double). Raw exponent = wrong answer.

Q6 Ripple-carry adder vs Carry-look-ahead adder

⚡ Core Intuition

Ripple-carry: each full adder waits for the previous carry → O(n) delay. CLA: compute every carry in parallel from inputs → O(log n) delay.

📖 Concept Explained
  • Full adder: Sum = A⊕B⊕Cin, Cout = AB + Cin(A⊕B).
  • Ripple: n full adders chained; carry propagates serially.
  • CLA: define Generate Gᵢ = AᵢBᵢ, Propagate Pᵢ = Aᵢ⊕Bᵢ.
  • Carry recurrence: Cᵢ₊₁ = Gᵢ + Pᵢ·Cᵢ → expanded → all carries from C₀ in two gate levels.
  • CLA: faster but more gates & wider fan-in.
🔗 Connects to full adder propagation delay ALU design
📐 Exam Answer Structure
  • Full-adder block diagram.
  • Ripple-carry chain (4-bit).
  • CLA equations: G, P, recurrence, fully expanded carries.
  • Compare delay + gate count in a table.

Q7 Booth's multiplier — algorithm + worked example (−7 × 5)

⚡ Core Intuition

Booth scans the multiplier two bits at a time. A run of 1s like 0111 is treated as 1000 − 0001 — replacing many add-shifts with one subtract-then-shift. Handles signed numbers natively.

📖 Concept Explained
  • Registers: A (accumulator, init 0), Q (multiplier), Q₋₁ (extra bit, init 0), M (multiplicand), counter = n.
  • Each step inspects pair (Q₀, Q₋₁):
  • 10 → A ← A − M (start of 1-run).
  • 01 → A ← A + M (end of 1-run).
  • 00 or 11 → no add/sub.
  • Then arithmetic right shift the {A, Q, Q₋₁} block. Repeat n times.
🔗 Connects to 2's complement shift-and-add carry-save multiplier
📐 Exam Answer Structure
  • State Booth's recoding insight.
  • Draw the flowchart.
  • Trace table with columns A, Q, Q₋₁, count.
  • Convert binary product back to decimal.

Q8 Restoring vs Non-restoring division algorithms

⚡ Core Intuition

Both subtract divisor from a running remainder. Restoring undoes the subtraction when remainder goes negative. Non-restoring keeps the negative and adds next time — saving a step.

📖 Concept Explained
  • Registers: A (remainder, 0), Q (dividend → quotient), M (divisor).
  • Restoring: shift, subtract; if A<0 → restore (A ← A+M) and set Q₀=0; else Q₀=1.
  • Non-restoring: shift; if A≥0 subtract M else add M; set Q₀ accordingly; skip restoration.
  • End-correction: if final A<0 → A ← A + M (last fix-up).
🔗 Connects to long division shift-subtract remainder/quotient

Q1 Hardwired vs Microprogrammed Control Unit

⚡ Core Intuition

Hardwired CU = fast custom circuit, hard to change. Microprogrammed CU = software-like control words stored in ROM, slower but flexible. RISC chose hardwired, CISC chose microprogrammed.

📖 Concept Explained
  • Control Unit generates timing & control signals that steer the datapath.
  • Hardwired: signals built from fixed combinational logic (gates, flip-flops, decoders, counters).
  • Microprogrammed: each machine instruction expands into a sequence of microinstructions fetched from Control Memory.
  • Adding a new opcode = re-design gates (HW) vs add microcode (MP).
🔗 Connects to RISC vs CISC Instruction cycle Microinstruction format Control memory
📐 Exam Answer Structure
  • Define CU + its job (generate control signals).
  • Draw both block diagrams side-by-side.
  • Comparison table (≥7 rows).
  • Where used (RISC vs CISC examples).

Q2 Microprogrammed CU — internal design (CAR, CBR, sequencer, control memory)

⚡ Core Intuition

A microprogrammed CU is a tiny computer inside the CPU. Each machine instruction = a small program of microinstructions read from a ROM.

📖 Concept Explained
  • Control Memory (CM): ROM storing microinstructions (control words).
  • CAR (Control Address Register) = micro-PC; holds address of next μinstruction.
  • CBR (Control Buffer Register) = μ-IR; holds fetched microinstruction.
  • Sequencer: decides next CAR — increment, branch, map (from opcode), or subroutine.
  • Mapping logic: opcode → start address of microroutine.
🔗 Connects to Control memory Horizontal vs vertical microinstruction Nanoprogramming
📐 Exam Answer Structure
  • Define microprogramming (Wilkes, 1951).
  • Draw block diagram with all 5 blocks labelled.
  • Explain flow: opcode → map → CAR → CM → CBR → signals.
  • Horizontal vs vertical microinstruction (1-line each).
💡 Example

ADD R1, R2 opcode maps to microroutine: μI₁ MAR←PC, μI₂ MBR←M[MAR]; PC++, μI₃ IR←MBR, μI₄ R1←R1+R2.

Horizontal μinstruction: wide, 1 bit per control signal, fast but large ROM. Vertical: narrow, encoded fields decoded by extra logic, smaller ROM but slower.

CAR drives, CM stores, CBR fires — CAR addresses, Control Memory stores, CBR fires control signals.

✅ Model Exam Answer
  1. Definition: microprogramming proposed by M.V. Wilkes (1951) — control signals stored as microinstructions in ROM.
  2. Diagram: block diagram (above).

Q2 Derive the pipeline speedup formula & solve a numerical

⚡ Core Intuition

📖 Concept Explained

  • As n → ∞, S → k (ideal limit).
  • Hazards & bubbles reduce real-world S below k.

Q1 Draw the memory hierarchy. Order it by cost, size, and speed.

⚡ Core Intuition

Fast memory is small and expensive; cheap memory is huge and slow — so we stack them.

📖 Concept Explained
  • CPU needs near-zero latency — registers + L1 cache deliver this.
  • Large programs need GB of space — DRAM + disk supply this.
  • Hierarchy keeps hot data near CPU, cold data far away.
  • Each upper level acts as a cache for the level below.
🔗 Connects to locality cache virtual memory EAT
💡 Example

Register access ≈ 1 ns; L1 ≈ 1 ns; L2 ≈ 4 ns; DRAM ≈ 100 ns; SSD ≈ 100 µs; HDD ≈ 10 ms.

"Registers Cache Main Secondary Tape" → "Really Clever Memory Saves Time".
✅ Model Exam Answer
  1. Definition: hierarchy = layered memory exploiting locality + cost.
  2. Draw pyramid: Reg → L1/L2/L3 → DRAM → SSD/HDD → Tape.
  3. Top: fastest, smallest, costliest per byte; bottom: opposite.
  4. Each upper level caches a subset of the level below.
  5. Goal: appear as fast as top, as large as bottom.

📚 Primary: unit 4 loc notes.pdf · Std: Mano, Stallings.

Q2 Explain memory interleaving. Compare low-order and high-order interleaving.

⚡ Core Intuition

Split memory into N banks so N reads happen in parallel — N× bandwidth, same chip speed.

📖 Concept Explained
  • One DRAM bank is busy during access cycle — others sit idle.
  • Interleaving = consecutive addresses live in different banks.
  • CPU issues addresses A, A+1, A+2 — all hit different banks → no waiting.
  • Low-order interleave: lowest k bits select bank (great for sequential).
  • High-order interleave: highest k bits select bank (great for fault-isolation).
🔗 Connects to DRAM banks bandwidth pipelining
💡 Example

Array A[0..15] on 4 low-order banks: reads of A[0..3] hit banks 0,1,2,3 in parallel → 4× speed-up.

"Low = Lane (parallel lanes), High = House (one block per house)."
✅ Model Exam Answer
  1. Definition: split memory into independent banks accessed in parallel.
  2. Draw 4-bank diagram (low-order and high-order side-by-side).
  3. Low-order: k LSBs select bank → consecutive addresses in different banks.
  4. High-order: k MSBs select bank → consecutive addresses inside one bank.
  5. Low boosts sequential bandwidth; high gives fault isolation.

📚 Primary: unit 4 loc notes.pdf · Mano "Computer System Architecture" Ch 12.

Q3 What is cache memory? Define locality of reference (temporal + spatial).

⚡ Core Intuition

Programs re-touch the same data (temporal) and nearby data (spatial). Cache exploits both.

📖 Concept Explained
  • Cache = small fast SRAM between CPU and DRAM.
  • Holds copies of frequently-used blocks of main memory.
  • Hit = data in cache (fast). Miss = fetch from DRAM (slow).
  • Temporal locality: recently accessed → likely accessed again soon (loop counter).
  • Spatial locality: nearby address → likely accessed next (array scan, instruction stream).
  • Cache works only because real programs show locality.
🔗 Connects to SRAM vs DRAM block size EAT prefetch
💡 Example

Iterating A[0..1023] with 64-byte blocks: every 16th access is a miss → 93.75% hit ratio purely from spatial locality.

"Time = same data again; Space = neighbour next."
✅ Model Exam Answer
  1. Cache = small fast SRAM between CPU and DRAM holding hot blocks.
  2. Draw CPU↔Cache↔Memory diagram with latencies.
  3. Temporal: same address re-accessed soon (loop variable).
  4. Spatial: nearby address accessed next (array element / next instruction).
  5. Cache only works because real workloads show both.

📚 Primary: unit 4 loc notes.pdf · Stallings "Computer Org" Ch 4.

Q4 Explain the three cache mapping functions with diagrams and worked example.

⚡ Core Intuition

Mapping answers: "where in the cache can a memory block live?" Direct = one slot. Fully = any slot. Set = one set, any line inside.

📖 Concept Explained
  • Main memory is divided into blocks; cache into lines of the same size.
  • The physical address is split into Tag | Line/Set | Word fields.
  • The split changes per mapping scheme — that's the whole question.
🔗 Connects to block size conflict miss comparator cost replacement

(a) Direct mapping

Each main-memory block maps to exactly one cache line: line = block mod cacheLines.

(b) Fully associative mapping

A block may sit in any cache line. Address = TAG | WORD (no index).

(c) Set-associative (N-way)

Cache split into S sets of N lines each. Block maps to one set; inside the set it can sit in any way.
WORKED EXAMPLE

Given: 64 KB cache, 4 KB block, 1 MB main memory. Find Tag / Line / Word for direct, fully associative, and 2-way set assoc.

Setup
  Main memory size = 1 MB     = 2^20  → 20 address bits
  Block (line) size = 4 KB    = 2^12  → 12 word bits
  Cache size        = 64 KB   = 16 lines of 4 KB

(a) DIRECT
  cacheLines = 64 KB / 4 KB = 16        → index = 4 bits
  word                                   = 12 bits
  tag = 20 − 12 − 4                      = 4 bits
  Address layout: | TAG 4 | LINE 4 | WORD 12 |

(b) FULLY ASSOCIATIVE
  No index — every block compared
  word = 12 bits
  tag  = 20 − 12                         = 8 bits
  Address layout: | TAG 8 | WORD 12 |

(c) 2-WAY SET ASSOCIATIVE
  sets = cacheLines / 2 = 16 / 2 = 8     → set = 3 bits
  word                                    = 12 bits
  tag  = 20 − 12 − 3                      = 5 bits
  Address layout: | TAG 5 | SET 3 | WORD 12 |
💡 Example

Direct: block-17 always lands in line 1 (17 mod 16). 2-way: block-17 maps to set 1 — either way 0 or way 1.

"Direct = Dorm room (one fixed bed). Fully = Freefall (any seat). Set = Shelf (row picked, slot free)."
✅ Model Exam Answer
  1. Define block / line / mapping function.
  2. Direct: one line per block; addr = TAG | LINE | WORD; cheapest, conflict-prone.
  3. Fully assoc: any line; addr = TAG | WORD; N comparators; no conflict miss.
  4. Set assoc: set fixed, way free; addr = TAG | SET | WORD; tradeoff.
  5. Show 3 address-bit splits using 64 KB / 4 KB / 1 MB numbers above.

Q6 Compare write-through and write-back policies.

⚡ Core Intuition

When CPU writes — does cache update memory now (through) or later (back)? "Now" is safe but slow; "later" is fast but risky.

📖 Concept Explained
  • Write-through: every CPU write goes to cache AND main memory.
  • Write-back: write only to cache; main memory updated when line is evicted.
  • Write-back needs a dirty bit per line.
  • On miss: write-allocate (fetch + write) vs no-write-allocate (skip cache).
  • Typical pairings: write-through + no-allocate; write-back + write-allocate.
🔗 Connects to dirty bit coherence bus bandwidth multiprocessor
💡 Example

Loop x++; 1000 times on a cached line: write-through = 1000 memory writes; write-back = 1 (on eviction).

"Through: write travels through to RAM. Back: write held back until eviction."
✅ Model Exam Answer
  1. State both definitions in one line each.
  2. Draw side-by-side diagram with CPU/Cache/Memory arrows.
  3. Compare: bus traffic, write latency, dirty bit, coherence.
  4. Mention write-allocate vs no-write-allocate.
  5. When to use: write-through for safety/multi-core, write-back for performance.

Q7 Derive Effective Access Time (EAT). Solve a numerical with hit ratio 0.9.

⚡ Core Intuition

Expected access time = (hit time, weighted by hit prob) + (miss time, weighted by miss prob). That's the whole formula.

📖 Concept Explained
  • h = hit ratio (fraction of accesses found in cache).
  • Tc = cache access time.
  • Tm = main memory access time.
  • Two common forms:
    • Hit-only-on-cache: EAT = h·Tc + (1−h)·(Tc + Tm)
    • Or simplified: EAT = h·Tc + (1−h)·Tm
  • Either is acceptable — be explicit which you used.
🔗 Connects to hit ratio cache levels AMAT 
EAT  =  h · Tc  +  (1 − h) · (Tc + Tm)
     =  Tc  +  (1 − h) · Tm            (always pay Tc; miss penalty = Tm)

Multi-level extension (L1 + L2):
EAT = h1·T1 + (1−h1)·h2·(T1+T2) + (1−h1)(1−h2)·(T1+T2+Tm)
WORKED EXAMPLE
Given:  Tc = 10 ns,  Tm = 100 ns,  h = 0.90

Method 1 (always pay Tc):
  EAT = Tc + (1 − h) · Tm
      = 10 + (1 − 0.90) · 100
      = 10 + 0.10 · 100
      = 10 + 10
      = 20 ns

Method 2 (split form):
  EAT = h · Tc + (1 − h) · (Tc + Tm)
      = 0.90 · 10 + 0.10 · 110
      = 9 + 11
      = 20 ns  ✓ same answer

Speed-up vs no cache:
  Without cache: every access = 100 ns
  With cache:    20 ns
  Speed-up = 100 / 20 = 5×
💡 Example

Hit ratio 0.95, Tc=2 ns, Tm=80 ns → EAT = 2 + 0.05·80 = 6 ns.

"EAT = Easy + Average + Tax. Tax = (miss rate) × (memory penalty)."
✅ Model Exam Answer
  1. Define hit ratio h, Tc, Tm.
  2. State formula: EAT = h·Tc + (1−h)·(Tc+Tm).
  3. Substitute given values, show 3 steps to numerical answer.
  4. Compute speed-up = Tm / EAT.
  5. Extend to multilevel cache (L1 + L2 form).

📚 Primary: unit 4 loc notes.pdf · Mano Ch 12.5.

Q8 Discuss the cache size vs block size tradeoff. Why does miss rate eventually rise as block size grows?

⚡ Core Intuition

Bigger blocks pull in more neighbours (good — spatial locality). But too big means fewer blocks fit (bad — capacity miss).

📖 Concept Explained
  • Small block: misses frequent — spatial locality wasted.
  • Medium block: sweet spot — spatial locality captured.
  • Large block: cache holds few blocks → more capacity / conflict misses.
  • Plus large block = longer transfer time on miss (miss penalty up).
  • Typical sweet spot: 32–128 bytes for L1.
🔗 Connects to 3 Cs (compulsory, capacity, conflict) spatial locality
💡 Example

A 16 KB cache with 256 B blocks holds only 64 blocks — any working set > 64 blocks thrashes. Same cache with 64 B blocks holds 256 — much better.

"Small = Skips neighbours; Big = Blocks compete."
✅ Model Exam Answer
  1. Define block size + cache size.
  2. Small block: poor spatial locality use → high miss rate.
  3. Increasing block: pulls in neighbours → miss rate falls.
  4. Beyond optimum: fewer blocks fit, miss penalty up → miss rate rises.
  5. Draw U-shaped curve; mark sweet spot 32–128 B for L1.

Red Flags — Common Penalties

Avoid these — each one costs 1–2 marks even when answer is "correct".

  • Forgetting IEEE 754 bias — you must add 127 (single) / 1023 (double) to actual exponent before storing.
  • Wrong cache bit-split — reversing tag/line/word order, or using #blocks instead of #lines.
  • Missing pipeline fill time — using k·n instead of k + n − 1 in the denominator.
  • Drawing Von Neumann without bus — lose marks if address/data/control buses aren't labelled separately.
  • Booth without Q₋₁ = 0 — not appending the implicit bit ruins the recoding table.
  • Confusing 1's and 2's complement — 1's has two zeros (+0 / −0); 2's has one.
  • SRAM vs DRAM swap — SRAM = flip-flop (cache), DRAM = capacitor (main memory).
  • Flynn's misclassified — multi-core CPU is MIMD, not SIMD; GPU shader is SIMD.
  • EAT without (1−h) term — forgetting the miss path makes the answer numerically wrong.
  • No diagram in 8-mark answer — auto-cap at 5/8 even with correct text.
  • Hardwired CU = "uses ROM" — reversed; ROM is microprogrammed. Hardwired = gates+FSM.
  • Calling cycle-stealing the same as burst DMA — burst hogs the bus, cycle-stealing releases it each word.
Final pitfall — Read the question twice. Many lose marks by answering “explain” when “derive” was asked.
