Q1 Evolution of wireless: 1G → 5G

⚡ Core Intuition

Each generation traded analog circuits for digital packets, multiplying speed roughly 10× per decade.

📖 Concept Explained
  • 1G (1980s): analog voice, FDMA, AMPS — no SMS, no security.
  • 2G (1990s): digital voice, TDMA/CDMA, SMS — GSM dominant.
  • 2.5G GPRS / 2.75G EDGE: packet data add-on, ~100 kbps.
  • 3G (2000s): UMTS/CDMA2000, ~2 Mbps, video calls.
  • 4G LTE (2010s): all-IP, OFDMA, ~100 Mbps–1 Gbps.
  • 5G (2020s): mmWave + massive MIMO, ~10 Gbps, low latency.
🔗 Connects to GSM CDMA OFDMA IoT
📐 Exam Answer Structure
  • Define wireless generation.
  • Draw labelled timeline (5 rectangles).
  • Table: gen / access / data-rate / use-case.
  • One-line transition driver per gen.
💡 Example

Your father's 1995 Nokia ran GSM 2G voice + SMS; today's iPhone uses 5G NR with carrier aggregation.

🔑 "A-D-P-I-M" — Analog, Digital, Packet, IP, MIMO (1G→5G).

✅ Model Exam Answer
  1. Define wireless generation = standardised radio-access technology era.
  2. Draw timeline (above).
  3. List per-gen: access, rate, service in tabular form.
  4. State driver: voice→data→video→IoT.
  5. Conclude: each gen reduces latency and increases spectral efficiency.

Q2 GSM architecture — MS to MSC call flow

⚡ Core Intuition

GSM is a structured phone tree: the handset talks to a tower, the tower's controller talks to a switch, the switch checks databases.

📖 Concept Explained
  • MS — Mobile Station (handset + SIM).
  • BTS — Base Transceiver Station (the antenna/tower).
  • BSC — Base Station Controller (manages many BTS, handoff).
  • MSC — Mobile Switching Centre (routes calls, switches).
  • HLR — Home Location Register (permanent subscriber DB).
  • VLR — Visitor Location Register (roamers near MSC).
  • EIR — Equipment Identity Register (IMEI blacklist).
  • AUC — Authentication Centre (secret keys, ciphering).
🔗 Connects to 2G SIM handoff PSTN
📐 Exam Answer Structure
  • Define GSM = circuit-switched digital cellular (2G).
  • Draw 3-subsystem block diagram (MS / BSS / NSS).
  • Expand each acronym in 1 line.
  • Trace call flow MS → BTS → BSC → MSC → HLR → PSTN.
💡 Example

Delhi → Mumbai call: handset (MS) → tower (BTS) → BSC → MSC checks HLR/AUC → routes via PSTN to Mumbai MSC.

🔑 "My Brother's Big Mac Has Very Easy Access" → MS, BTS, BSC, MSC, HLR, VLR, EIR, AUC.

✅ Model Exam Answer

Q3 CDMA — spread spectrum & code orthogonality

⚡ Core Intuition

Everyone speaks at once but in unique languages — only your friend with the right "code" understands you.

📖 Concept Explained
  • CDMA = Code Division Multiple Access (3G core).
  • Each user's bit is XOR-spread with a unique PN code (chips).
  • All users transmit on same frequency, same time.
  • Receiver multiplies by own code → recovers signal.
  • Orthogonal codes: dot-product = 0 → no interference.
  • Walsh codes used for orthogonality in IS-95.
🔗 Connects to 3G IS-95 Walsh spread spectrum
📐 Exam Answer Structure
  • Define CDMA + spread spectrum.
  • Diagram: data × code = chip-stream.
  • Numerical: 2-user orthogonal decode.
  • Advantages: capacity, security, soft handoff.
Numerical:
User A bit = +1, Code A = (+1,+1,-1,-1)
User B bit = -1, Code B = (+1,-1,+1,-1)

Sum on air = A·codeA + B·codeB
           = (+1,+1,-1,-1) + (-1,+1,-1,+1)
           = ( 0,+2,-2, 0)

Decode A: Sum · codeA  / 4
        = (0·1 + 2·1 + (-2)·-1 + 0·-1)/4
        = (0 + 2 + 2 + 0)/4 = +1   ✓  (matches A)
💡 Example

IS-95 / Reliance CDMA2000 used Walsh-64 codes — 64 users share same 1.25 MHz band.

🔑 "CDMA = Cocktail-party" — many languages, one room, you tune your ear.

✅ Model Exam Answer
  1. Define CDMA = multiple access via unique PN/Walsh codes.
  2. Draw data × code diagram.
  3. Show 2-user orthogonal numerical (above).
  4. Advantages: high capacity, soft-handoff, jam-resistant.
  5. Used in IS-95, CDMA2000, UMTS-WCDMA.

Q4 4G LTE features + OFDMA principle

⚡ Core Intuition

Instead of one fat pipe, OFDMA splits the band into many thin orthogonal subcarriers — each user grabs a few.

📖 Concept Explained
  • 4G = LTE / LTE-Advanced — all-IP, no circuit switching.
  • Downlink: OFDMA. Uplink: SC-FDMA (lower PAPR).
  • Peak rate ~100 Mbps mobile, ~1 Gbps stationary.
  • MIMO 2×2 / 4×4 antennas → spatial multiplexing.
  • Latency ~10 ms (vs 3G ~100 ms).
  • Services: VoLTE, HD video, IoT (NB-IoT add-on).
🔗 Connects to OFDM MIMO VoLTE 5G NR
📐 Exam Answer Structure
  • Define 4G = all-IP packet network.
  • OFDMA subcarrier diagram.
  • List 5 features (rate, MIMO, latency, IP, OFDMA).
  • Compare with 3G in 2 lines.
💡 Example

Jio 4G LTE band 40 (TD-2300) uses 20 MHz with 1200 subcarriers spaced 15 kHz.

🔑 "4G = OMIPV" — OFDMA, MIMO, IP-only, <10 ms latency, VoLTE.

✅ Model Exam Answer

Q6 Bluetooth piconet & scatternet topology

⚡ Core Intuition

One Bluetooth master leads up to 7 slaves (piconet); multiple piconets sharing a node form a scatternet.

📖 Concept Explained
  • Piconet = 1 master + up to 7 active slaves.
  • All share master's frequency-hopping clock (1600 hops/s).
  • Up to 255 parked slaves (low-power, addressable).
  • Scatternet = ≥2 piconets sharing a bridge node.
  • Bridge can be slave/slave or master/slave.
  • Operates at 2.4 GHz ISM, 79 channels of 1 MHz.
🔗 Connects to 802.15.1 FHSS PAN

💡 Example

🚨 Common mistakes (red flags)

  • Don't confuse GSM with GPRS — GPRS is a 2.5G packet-data add-on over GSM, not a separate generation.
  • CDMA is NOT a single standard — IS-95 (2G), CDMA2000 (3G), WCDMA (3G UMTS) all use CDMA principle.
  • 4G uplink uses SC-FDMA, not OFDMA — lower PAPR for battery saving.
  • Bluetooth piconet has 7 active + up to 255 parked slaves — examiners often ask the parked count.
  • Don't mix HLR (home, permanent) with VLR (visiting, temporary) — they store different data.
  • WLL ≠ WLAN — WLL replaces the wired local loop (last mile), WLAN is in-building.
  • 1G was analog — writing "1G digital" is an instant mark cut.

Q1 Cellular concept, hexagonal geometry & frequency reuse

⚡ Core Intuition

Recycle the same frequencies in cells far apart enough that they don't hear each other.

📖 Concept Explained
  • Spectrum is scarce — one big tower can't serve a city.
  • Split area into small cells; each gets a slice of channels.
  • Group of N cells using all channels once = cluster.
  • Same channel reused in next cluster — that's frequency reuse.
  • Hexagon chosen: tiles plane, closest to circle, equal-area neighbours.
  • Valid cluster sizes: N = i² + ij + j² → 1, 3, 4, 7, 9, 12, 13, 19…
🔗 Connects to Q2 reuse distance Q7 SIR Q5 cell splitting
📐 Exam Answer Structure
  • Define cell + cluster + reuse (1 line each).
  • Draw 7-cell cluster with co-channel cells marked.
  • List valid N values + give formula N = i²+ij+j².
  • State capacity = M·k·N⁻¹ (total channels recycled).
💡 Example

660 total channels, N = 7 → each cell gets 660/7 ≈ 94 channels. Across 10 clusters in city = 6600 simultaneous calls.

Definition
Frequency reuse = assigning same set of carrier frequencies to two cells whose mutual interference is acceptably low.
"Hex Cluster Recycle" — Hex tile, cluster of 7, recycle far away.
✅ Model Exam Answer
  1. Def: reuse = same channels in non-adjacent cells.
  2. Diagram: 7-cell hex cluster + adjacent cluster with marked co-channel cell.
  3. N values: 1,3,4,7,9,12,13,19 (i²+ij+j²).
  4. Capacity = (Spectrum ÷ N) × #clusters in service area.
  5. Why hex: tiles plane, uniform 6 neighbours, minimal overlap vs circle.

Q2 Derive Q = D/R = √(3N) & C/I formula

⚡ Core Intuition

Bigger cluster → co-channel cells further apart → less interference → cleaner reception, but fewer channels per cell.

📖 Concept Explained
  • R = cell radius (centre → vertex of hexagon).
  • D = distance between centres of two nearest co-channel cells.
  • Q = co-channel reuse ratio = D/R.
  • Geometric derivation on hex lattice: D² = 3N · R² ⇒ Q = √(3N).
  • Larger N → larger Q → higher SIR but lower capacity. Tradeoff.
  • For N=7: Q = √21 ≈ 4.58.
🔗 Connects to Q1 cluster Q7 SIR
📐 Exam Answer Structure
  • Define R, D, Q (1 line each).
  • State result Q = √(3N) — derivation in 3 lines.
  • Table of Q vs N (1,3,4,7,12).
  • Plug into SIR = (1/6)·Q^n where n = path-loss exponent.
  • Numerical: N=7, n=4 ⇒ SIR ≈ 18.66 dB ✓ (above 18 dB threshold).
D² = 3 N R²        (hex lattice geometry)
Q  = D / R = √(3N)

SIR  =  S / I  =  (1/6) · (D/R)^n
     =  (1/6) · (√3N)^n      [for 6 first-tier interferers]

Path-loss exponent n: free-space 2, urban 3-4.

Q vs N:
   N = 3   →  Q = 3.00
   N = 4   →  Q = 3.46
   N = 7   →  Q = 4.58
   N = 12  →  Q = 6.00
   N = 19  →  Q = 7.55
Given: N = 7, n = 4 (urban path-loss)
Find:  SIR in dB

Q   = √(3 × 7) = √21 ≈ 4.583
SIR = (1/6) × (4.583)^4
    = (1/6) × 441.3
    = 73.5
SIRdB = 10·log₁₀(73.5) ≈ 18.66 dB ✓ above 18-dB threshold

Answer: SIR ≈ 18.66 dB, acceptable for N = 7.
"Q is √3N" — say it three times. Students drop the √3 every year.
✅ Model Exam Answer
  1. State Q = D/R = √(3N), derive D² = 3NR².
  2. Draw reference cell + 6 first-tier co-channel cells.
  3. Show SIR = (Q^n)/6.
  4. Plug N=7, n=4 → SIR ≈ 18.66 dB.
  5. Conclude: N=7 is industry standard balance.

Q3 Handoff strategies — MAHO, MCHO, NCHO

⚡ Core Intuition

When a moving phone leaves a cell, the call must jump to the next cell without dropping. Who decides? — that's what differs.

📖 Concept Explained
  • Handoff = transferring call from cell A to cell B as MS moves.
  • NCHO (Network-Controlled): MSC decides — used in AMPS 1G. Slow (~10 s).
  • MAHO (Mobile-Assisted): MS measures & reports, BS/MSC decides — GSM 2G. ~1 s.
  • MCHO (Mobile-Controlled): MS measures & decides itself — DECT, some 3G. ~0.1 s.
  • Trigger: RSSI < threshold OR SIR drops OR distance exceeds.
  • Hard handoff (break-before-make) vs soft handoff (CDMA, make-before-break).

Q4 Channel assignment strategies — fixed, dynamic, hybrid

⚡ Core Intuition

Fixed = each cell gets pre-allocated channels. Dynamic = pool, give on demand. Hybrid = mix of both.

📖 Concept Explained
  • Fixed (FCA): cell owns its channels permanently. Simple. Wastes idle ones.
  • Borrowing: a busy cell borrows channels from neighbour if SIR allows.
  • Dynamic (DCA): MSC keeps a central pool; assigns on call request.
  • Hybrid (HCA): split — A% fixed per cell, B% dynamic pool.
  • DCA reduces blocking under uneven traffic; FCA simpler & lower signalling.

Q7 Co-channel & adjacent-channel interference, SIR

⚡ Core Intuition

Interference = unwanted signals on your channel (CCI) or leaking from neighbour channels (ACI). Both lower SIR.

🚨 Common mistakes (red flags)

  • Forgetting the √3 in Q = √(3N). Writing Q = √N or Q = 3N loses derivation marks.
  • Confusing hard / soft handoff (timing) with MAHO/MCHO/NCHO (control).
  • Claiming sectoring "adds channels". It only splits existing ones; gain is from improved SIR.
  • Using SIR formula with 6 interferers for sectored cells — use 2 (120°) or 1 (60°).
  • Confusing cluster size N with number of channels per cell.
  • Drawing cells as circles — must be hexagons; explain why (tiles plane, equal neighbours).
  • Forgetting Tx power must drop by (1/2)^n after cell splitting to keep SIR.
  • Mixing up Erlang B (cleared) and Erlang C (queued) — cellular uses B.
  • Writing valid cluster sizes as 1,2,3,4,5… — only i²+ij+j² values are valid (no N=2,5,6,8…).
  • Forgetting units: Erlangs are dimensionless (channel·hr / hr).

🚨 Common mistakes (prof penalties)

Q2 Explain the fixed network transmission hierarchy (T/E carriers, SDH).

─── CELLULAR GEOMETRY ───────────────────────────
Cluster size       N = i² + i·j + j²
                   (i, j = non-negative integers)
                   Valid N: 1, 3, 4, 7, 9, 12, 13, 19, 21 …

Reuse distance     D = R · √(3N)
Reuse ratio        Q = D / R = √(3N)
                   Larger Q → less interference, lower capacity

Cells per cluster  S = k · N
                   k = channels/cell, S = total channels

System capacity    C = M · S = M · k · N / N = M · k
                   M = number of clusters in coverage area

─── CO-CHANNEL INTERFERENCE ─────────────────────
Signal-to-interference (omni, 6 co-channel cells)
                   C/I = (1/6) · (D/R)^n
                       = (1/6) · (√(3N))^n
                       = (3N)^(n/2) / 6
                   n = path-loss exponent (typ. 3–4)

Express in dB:     C/I (dB) = 10 · log₁₀ (C/I)
Min acceptable     C/I ≥ 18 dB (AMPS), 9 dB (GSM)

─── CDMA / SPREAD SPECTRUM ──────────────────────
Processing gain    G = BW / Rb   (or  Gp = Tb / Tc)
                   BW = spread bandwidth, Rb = data rate
                   In dB:  G(dB) = 10·log₁₀(BW/Rb)

Chip rate (IS-95)  Rc = 1.2288 Mcps
                   G = 1.2288e6 / 9600 = 128  ≈ 21 dB

Jamming margin     M_j = G − [SNR_req + L_sys]

─── SHANNON CAPACITY ────────────────────────────
Channel capacity   C = B · log₂(1 + S/N)
                   C in bps, B in Hz, S/N linear (not dB)
Convert dB→linear  S/N = 10^(SNR_dB / 10)

─── ERLANG-B (BLOCKING) ─────────────────────────
                    A^N / N!
P_B  =  ─────────────────────────────
        Σ (k=0 to N)  A^k / k!

   A   = offered traffic (Erlangs) = λ · h
   N   = number of channels
   P_B = blocking probability
   Use Erlang-B table — don't compute by hand in exam.

─── ISDN BIT RATES ──────────────────────────────
B channel  = 64 kbps   (bearer / voice / data)
D channel  = 16 kbps (BRI) · 64 kbps (PRI)
BRI        = 2B + D   =  2·64 + 16  = 144 kbps  (user)
                       + 48 framing  = 192 kbps  (line)
PRI (US/JP)= 23B + D  = 23·64 + 64   = 1.544 Mbps (T1)
PRI (EU)   = 30B + D  = 30·64 + 64   = 2.048 Mbps (E1)

─── MISC ────────────────────────────────────────
Doppler shift      f_d = (v / λ) · cos θ
Free-space loss    L = 20·log(d) + 20·log(f) + 32.44  (dB, km, MHz)
GSM frame          1 TDMA frame = 8 slots = 4.615 ms

Red Flags · Common Mistakes

Examiner deducts for these. Re-read 2× before entering hall.

  • Writing Q = √(3·N) but forgetting the √. Always write √(3N), not 3N.
  • Mixing up i and j in N = i² + ij + j². Lowercase ij is a single product, NOT i·j².
  • ISDN BRI written as 23B + D. BRI = 2B + D. 23B+D is PRI (US/Japan).
  • FHSS confused with DSSS. FHSS hops carrier frequencies, DSSS multiplies by PN code.
  • Saying GSM uses CDMA. GSM = TDMA (+ FDMA between carriers). IS-95 is CDMA.
  • Calling soft handoff "GSM feature". Soft handoff = CDMA only. GSM does hard handoff.
  • Forgetting log base in Shannon. C = B · log₂(1 + S/N) — base 2, not 10.
  • Using SNR in dB inside Shannon directly. Convert to linear: S/N = 10^(dB/10).
  • Drawing GSM without HLR/VLR. NSS = MSC + HLR + VLR + AuC + EIR. Skipping registers loses 2 marks.
  • Writing "1G = digital". 1G is analog (AMPS). 2G is the first digital generation.
  • Confusing reuse ratio Q with cluster size N. Q = D/R (distance ratio), N = cells per cluster.
  • Hexagon sides not equal. Hexagonal cells must be regular hexagons — equal sides for valid geometry.
  • Forgetting near-far problem on CDMA. Always mention power control as the fix.
  • Writing "LTE is 3G". LTE = 4G. UMTS / WCDMA = 3G.
  • Skipping units. Bandwidth in Hz, data rate in bps, distance in km. Mark-loss otherwise.
