Q1 Define data. Explain data types and variables.

⚡ Core Intuition

Data = raw facts. Type = how you can compare them (label / order / distance / ratio).

📖 Concept Explained
  • Data: facts/observations collected for analysis.
  • Qualitative (categorical): nominal + ordinal.
  • Quantitative (numerical): discrete + continuous.
  • 4 scales: Nominal → Ordinal → Interval → Ratio (info increases).
🔗 Connects to measurement scales variables stats
💡 Example
  • Nominal: hair colour (black, brown) — no order.
  • Ordinal: rank (1st, 2nd) — order, no equal gap.
  • Interval: temp °C — equal gap, no true zero.
  • Ratio: weight kg — true zero, ratios meaningful.
Mnemonic: "NOIR" — Nominal → Ordinal → Interval → Ratio (info grows).

Q3 Explain types of data distributions.

⚡ Core Intuition

Distribution = shape of data when you plot how often each value appears.

📖 Concept Explained
  • Normal: symmetric bell — mean = median = mode.
  • Skewed: tail on one side (left/right).
  • Uniform: flat — all values equally likely.
  • Bimodal: two peaks — two sub-groups.

Q4 Difference between Database and Data Warehouse.

⚡ Core Intuition

Database = OLTP (transactional, real-time, normalised). Warehouse = OLAP (read-optimised historical archive). One feeds the other.

📖 Concept Explained
  • Database records current data; warehouse analyses historical data.
  • Database is normalised (complex joins); warehouse is denormalised (simple joins).
  • Database = ER-modelled; warehouse = dimensional (star/snowflake).
  • Warehouse is subject-oriented, integrated, time-variant, non-volatile.

Q6 Supervised vs Unsupervised learning (with reinforcement).

⚡ Core Intuition

Supervised = learn with labelled answer-key. Unsupervised = find groups with no answer-key. Reinforcement = learn via reward.

📖 Concept Explained
  • Supervised: input X + output Y given. Goal: learn f(X)=Y. Tasks: classification, regression.
  • Unsupervised: only X given. Goal: find structure. Tasks: clustering, association.
  • Reinforcement: agent acts in env, gets reward, learns policy.

Q2 Explain special values in R: NA, NULL, NaN, Inf.

⚡ Core Intuition

NA = missing. NULL = nothing. NaN = bad math. Inf = too big.

📖 Concept Explained
  • NA — "Not Available". A placeholder where a value should exist but does not.
  • NULL — empty / undefined object; has length 0.
  • NaN — "Not a Number"; result of 0/0 or log(-1).
  • Inf / -Inf — infinity; 1/0 → Inf, -1/0 → -Inf.
  • All propagate: any arithmetic with NA / NaN gives NA / NaN.
🔗 Connects to is.na() is.null() is.nan() is.finite()

Q4 Explain matrices and arrays. Create two 3×3 matrices and multiply them.

⚡ Core Intuition

Matrix = 2-D vector. Array = n-D vector. Same type throughout.

📖 Concept Explained

  • Indexing — m[i, j], m[i, ] = row, m[, j] = column.
  • Multiply — A * B = element-wise, A %*% B = matrix product.
  • apply() — apply(m, 1, fn) per row, 2 per column.

🚨 Red flags — examiner penalty list

Q7 Object-Oriented Programming in R — S3, S4 and Reference Classes

⚡ Core Intuition

R has three OOP systems, growing in formality.

S3 = informal (slap a class attribute). S4 = strict (formal slots). RC = Java/Python-style with mutable objects + methods inside class.

📖 Concept Explained
  • S3 — informal; create with class(obj) <- "name"; method dispatch via UseMethod().
  • S4 — formal; setClass() defines slots, new() creates objects, setGeneric/setMethod add methods. Access slots with @.
  • Reference Class (RC / R5) — setRefClass(); methods belong to class; objects are mutable (no copy-on-modify).
  • Methods in S3/S4 belong to generic functions; in RC they belong to the class.
🔗 Connects to class / objectgeneric functionspolymorphismencapsulation 
# ════════ S3 — informal ════════
s <- list(name="John", age=21, GPA=3.5)
class(s) <- "student"            # just an attribute

# generic + method
print.student <- function(obj) {
  cat("Name:", obj$name, "GPA:", obj$GPA, "\n")
}
print(s)         # dispatches to print.student

# ════════ S4 — formal ════════
setClass("student",
  slots = list(name="character", age="numeric", GPA="numeric"))
s <- new("student", name="John", age=21, GPA=3.5)
s@name           # access slot with @
s@GPA <- 3.7     # modify

setGeneric("show_student", function(x) standardGeneric("show_student"))
setMethod("show_student", "student", function(x) {
  cat("S4 student:", x@name, "\n")
})
show_student(s)

# ════════ Reference Class (RC) ════════
Student <- setRefClass("Student",
  fields  = list(name="character", GPA="numeric"),
  methods = list(
    greet = function() cat("Hi, I am", name, "\n"),
    bump  = function() { GPA <<- GPA + 0.1 }   # mutates self
  )
)
s <- Student$new(name="John", GPA=3.5)
s$greet()        # method belongs to class
s$bump()
s$GPA            # 3.6  (mutated in place)

🚨 Common mistakes / red flags

  • Confusing sapply (simplified output) with lapply (always a list). Examiners love this Q.
  • Using if on a whole vector — only first element checked + warning. Use ifelse() instead.
  • Forgetting repeat needs an if(...) break — infinite loop in exam program = 0 marks.
  • Mixing up S3 ($) and S4 (@) accessors.
  • Saying "RC follows copy-on-modify" — it does NOT; RC is mutable / pass-by-reference.
  • Writing switch without a default branch — silent NULL if no match.
  • Forgetting row.names = FALSE in write.csv() — extra junk column appears in CSV.
  • Calling lists "vectors" — vectors are homogeneous, lists are heterogeneous.
  • In S4, missing setGeneric() before setMethod() — method won't dispatch.
  • Using &&/|| on a long vector — only the first element is tested.

Q8 Statistical Distributions in R — d / p / q / r Prefix Family

🚨 Common mistakes / red flags

  • Forgetting na.rm=TRUE — mean(c(1,2,NA)) returns NA, not 1.5.
  • Using = instead of <- inside a function call as assignment (works but unidiomatic).
  • Writing mode(x) — that's a type checker, not statistical mode. R has no built-in mode.
  • Confusing dnorm (height) with pnorm (cumulative) — answers different questions.
  • In cor.test() assuming p-value alone proves causation — only shows linear association.
  • Forgetting library(ggplot2) before ggplot() — error "could not find function".
  • Mis-reading chisq.test() warning — expected counts < 5 makes χ² unreliable.
  • Missing set.seed() before rnorm() → answers differ on re-run.
  • Forgetting row.names=FALSE in write.csv() — leaks an extra index column.
  • Not closing dev.off() after png() — file stays open / blank.
  • Leaving debug(f) active — every call drops you into the debugger; remember undebug(f).

Red Flags — Common Mistakes

Each one is a guaranteed mark-loss. Fix in your head now.

  • NA handling: mean(x) returns NA if any value is missing — always pass na.rm = TRUE or pre-clean with na.omit().
  • NA ≠ NULL: is.na(NULL) returns logical(0), not TRUE. NA is a value; NULL is absence.
  • Factor levels: never compute mean/sum on a factor; convert with as.numeric(as.character(f)) — direct as.numeric(f) returns level codes, not values.
  • String → factor surprise: pre R 4.0, read.csv auto-converted strings to factors; always set stringsAsFactors = FALSE to be explicit.
  • sapply vs lapply: sapply tries to simplify and may return a matrix when you expected a vector — use vapply for type safety in production code.
  • $ on wrong type: $ works on lists and data.frames, NOT on vectors, matrices, or S4 objects. S4 uses @.
  • 1-based indexing: x[0] in R returns an empty vector, not the first element. First element is x[1].
  • Vector recycling: c(1,2,3,4) + c(10,20) silently recycles to c(11,22,13,24); warns only on length mismatch, not on multiples.
  • apply MARGIN: MARGIN = 1 = rows, MARGIN = 2 = columns. Reversed is a classic exam trap.
  • ggplot vs qplot: ggplot needs a data.frame, not a vector; passing a vector throws a cryptic error about aes().
  • d vs p prefix: dnorm = density at a point (PDF), pnorm = cumulative P(X≤x) (CDF). Mixing these up costs whole marks.
  • S3 dispatch: forgetting UseMethod("fn") in the generic — without it, no method dispatch happens.
  • Lazy library load: using ggplot2 / dplyr functions without library(ggplot2) → "could not find function" error.
  • Hard-coded paths: use setwd() or relative paths; absolute Windows paths with single \ break — escape as \\ or use /.
