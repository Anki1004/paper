Q1 Edward de Bono's Six Thinking Hats

⚡ Core Intuition

Six coloured hats = six lenses; wear one at a time so a group thinks together instead of arguing past each other.

📖 Concept Explained
  • Proposed by Edward de Bono (1985) for parallel thinking.
  • Each hat = one mode; switch hats, not minds.
  • Prevents tunnel vision and ego-driven debate.
  • Used in meetings, decisions, group problem solving.
🔗 Connects to problem solving critical thinking group decision creativity
📐 Exam Answer Structure
  • Define + author + year (1 line).
  • Draw 6 coloured hats with one-line role each.
  • Sample question each hat asks.
  • Importance: balance, objectivity, creativity.
HatRoleSample question
WhiteFacts, data, infoWhat do we know?
RedEmotions, intuitionHow do I feel?
BlackCaution, risks, judgementWhat can go wrong?
YellowOptimism, benefitsWhat is the upside?
GreenCreativity, alternativesWhat new idea fits?
BlueProcess, controlWhat's the next step?
💡 Example

Team decides on a new app feature: White lists user data, Red shares gut reactions, Black flags bugs, Yellow estimates revenue, Green pitches alt designs, Blue books the next meeting.

🧠 Mnemonic — "WReBYGB" → "Wrebygb" = White, Red, Black, Yellow, Green, Blue. Or: "Facts, Feelings, Faults, Favours, Fresh, Flow."
✅ Model Exam Answer
  1. Define: Six Thinking Hats is a parallel-thinking method by Edward de Bono (1985).
  2. Draw the 6-hat diagram with colours.
  3. List each hat with role + sample question (table).
  4. State importance: balanced perspective, removes ego, separates emotion from logic.
  5. Example: group decision on a new product launch.

Q2 Inductive vs Deductive Reasoning

⚡ Core Intuition

Induction = bottom-up (cases → rule). Deduction = top-down (rule → case). Induction is probable; deduction is certain (if premises hold).

📖 Concept Explained
  • Inductive: specific observations → general conclusion.
  • Deductive: general premises → specific conclusion.
  • Induction builds theories; deduction tests them.
  • Induction = probable. Deduction = valid if premises true.
🔗 Connects to logic inference syllogism hypothesis
📐 Exam Answer Structure
  • Define both, side-by-side.
  • Draw two-flow diagram (bottom-up vs top-down).
  • Comparison table (5–6 rows).
  • One worked example each (syllogism for deductive).
AspectInductiveDeductive
DirectionSpecific → GeneralGeneral → Specific
CertaintyProbableCertain (if premises true)
UseHypothesis, predictionProof, validation
RiskMay be wrong with new dataFails if premise is false
ToolsObservation, patternSyllogism, formal logic
Example"This coffee shop always has a line.""All humans are mortal; Socrates is human; Socrates is mortal."
💡 Example — Inductive

Visited shop 5 mornings, line every time → induce: "this shop is always busy in the morning." (Not guaranteed; the 6th visit may differ.)

💡 Example — Deductive (syllogism)

P1: All seats are reserved for ticket holders. P2: I don't have a ticket. Conclusion: I have no reserved seat. (Valid: premises force conclusion.)

🧠 Mnemonic → "In-duction is In-formation building; De-duction is De-rivation." Sherlock Holmes mostly used induction despite his name.
✅ Model Exam Answer
  1. Define both with one-line direction (specific→general vs general→specific).
  2. Draw the dual-flow diagram (bottom-up vs top-down).
  3. Comparison table with 5–6 rows.
  4. One worked example per type (use the Socrates syllogism for deductive).
  5. State role in critical thinking: induction → hypothesis; deduction → validation.

📚 Primary: Inductive and Deductive Reasoning.docx · Supplemented: standard logic textbook conventions.

Q3 The Rhetorical Situation — Purpose, Audience, Topic, Context

⚡ Core Intuition

Every message is shaped by four forces: why (purpose), who (audience), what (topic), where/when (context).

📖 Concept Explained
  • The rhetorical situation = the conditions a communicator works within.
  • Coined by Lloyd Bitzer (1968); standard frame for analysis.
  • Change any one element → message must change.
  • Used to plan speeches, essays, ads, posts.
🔗 Connects to rhetorical strategies ethos/pathos/logos audience analysis
📐 Exam Answer Structure
  • Define rhetorical situation (1 line).
  • Draw the 4-quadrant model.
  • Explain each element with sub-points.
  • Worked mini-example (one topic across two contexts).
ElementDefinitionSub-factors
PurposeWhy the message exists.Inform, persuade, entertain, educate
AudienceWhom the message targets.Target, intended, unintended, imagined
TopicSubject / issue addressed.Content, tone, urgency, appeals
ContextSurrounding circumstances.Medium, time, place, culture, constraints
💡 Example

Topic = climate change. To scientists (audience) at a conference (context), purpose = inform → use logos + data. To children at school → purpose = educate → use story + visuals.

🧠 Mnemonic — "PATC" → "Please Address The Crowd" = Purpose, Audience, Topic, Context.

Q4 Critical Thinking — Definition & Four Types (Analysis, Communication, Inference, Observation)

⚡ Core Intuition

Critical thinking = think clearly about thinking. Four pillars hold it up: observe facts, infer meaning, analyse parts, communicate clearly.

📖 Concept Explained
  • Active, persistent reasoning to decide what to believe / do.
  • Dewey: thinking with reasons. Ennis: reflective + reasonable.
  • Resists bias, surface judgements, and emotional reactions.
  • Foundation of decision-making, debate, problem solving.
🔗 Connects to problem solving reasoning rhetoric decision making

Doctor sees a rash (observation) → guesses allergy (inference) → checks history & tests (analysis) → explains diagnosis to patient (communication).

Q6 Rhetorical Devices — Alliteration & Amplification

⚡ Core Intuition

Alliteration = repeat the same starting sound for rhythm. Amplification = repeat / expand an idea for emphasis.

📖 Concept Explained
  • Both are devices, not strategies — they shape sound and weight.
  • Alliteration → musicality & memorability.
  • Amplification → builds persuasion through repetition.
  • Common in speeches, slogans, poetry.
🔗 Connects to rhetorical strategies pathos speech writing poetry
📐 Exam Answer Structure
  • Define each device (1 line).
  • Quote 2 literary examples per device.
  • List purposes (emphasis, rhythm, persuasion, imagery).
  • End with one combined example.
Definition — Alliteration: a literary device in which words that start with the same sound are repeated in close proximity.
"She sells seashells by the seashore."
"The five fearless firefighters fought the flames."
"The busy bees buzzed around the bright blue flowers."
Definition — Amplification: a literary device in which a statement or idea is repeated or expanded upon to emphasise its importance.
"We shall fight on the beaches, we shall fight on the landing grounds, we shall fight in the fields and in the streets." — Winston Churchill
"The impact of climate change is not just environmental, it's also economic, social, and moral."
DeviceWhat repeatsEffect
AlliterationStarting soundRhythm, musicality, memory
AmplificationIdea / phrase (expanded)Emphasis, persuasion, vividness
Common purposes: (1) create emphasis (2) add rhythm/musicality (3) build persuasion (4) create vivid imagery.
🧠 Mnemonic → "Alliteration = Audible. Amplification = Abundant." One repeats sound, the other repeats meaning.
✅ Model Exam Answer
  1. Define alliteration; give 2 literary examples.
  2. Define amplification; give Churchill + climate example.
  3. Common purposes (4 bullets).
  4. Contrast: sound vs idea repetition.

Q7 Problem Solving in Critical Thinking

⚡ Core Intuition

Problem solving = pipeline from understanding the issue to reflecting on the fix — not just guessing solutions.

📖 Concept Explained
  • Applies logic + creativity + evaluation to find best solution.
  • Tied to IDEAL / Polya 4-step model in classic theory.
  • Iterative — review feeds the next problem.
  • Used in engineering, management, daily life.
🔗 Connects to critical thinking Six Hats decision making
📐 Exam Answer Structure
  • Define problem solving.
  • Draw 7-step (or IDEAL 5-step) flow.
  • 1-line explanation per step.
  • Importance: decisions, innovation, resilience.
StepWhat you do
1. UnderstandDefine problem; ask questions; break down components.
2. Gather informationResearch, observe, collect perspectives.
3. Identify solutionsBrainstorm; list alternatives; weigh pros / cons.
4. Evaluate / analyseAssess feasibility, risk, outcome; remove bias.
5. DecideChoose the most effective option.
6. ImplementAct — apply the chosen solution.

Falling app sales: define drop (1) → gather analytics (2) → brainstorm features / pricing / marketing (3) → evaluate cost vs impact (4) → pick referral feature (5) → ship it (6) → measure retention next month (7).

Q8 Observation vs Inference

⚡ Core Intuition

Observation = what you see. Inference = what you conclude from it. Mixing the two is the #1 critical-thinking error.

📖 Concept Explained
  • Observation is factual, objective, sense-based.
  • Inference is interpretive, evidence-based, tentative.
  • Good inference rests on accurate observation.
  • Inferences can be inductive or deductive.
🔗 Connects to analysis inductive/deductive hypothesis
📐 Exam Answer Structure
  • Define both side-by-side.
  • Show one example pair.
  • Comparison table (4–5 rows).
  • Note: inference is tentative.
AspectObservationInference
NatureFactual, sense-basedInterpretive, reasoned
CertaintyDirect & objectiveTentative & probable
ToolsSenses, measurementLogic, prior knowledge
Example"Dark clouds in the sky.""A storm is coming."
RiskInaccurate if biasedWrong if observation is flawed
Pitfall: "A person walking toward the door is leaving." → walking is observation; leaving is an inference. Don't merge them.
🧠 Mnemonic → "See first, conclude second." Observation comes before inference, always.
✅ Model Exam Answer
  1. Define both (1 line each).
  2. Diagram: observation → inference.
  3. Comparison table (5 rows).
  4. Pitfall: don't merge fact with conclusion.

🚨 Common mistakes (red flags)

  • Inductive vs Deductive — Inductive goes specific → general, deductive goes general → specific. Do not swap them.
  • Inference is not observation — "Storm is coming" is inference; "Dark clouds" is observation.
  • Don't mix strategy with device — Compare/Contrast/Classify/Describe = strategies. Alliteration/Amplification = devices.
  • Six Hats colour-meaning — Black ≠ negative person; it means caution. Red = emotion, not anger.
  • Rhetorical situation — four elements, not three. Don't drop "Context".
  • Ethos / Pathos / Logos — credibility, emotion, logic — in that order. Don't swap with hats.
  • Definition first — every long answer must start with a 1-line definition, not history.

Q1 Explain the CRAAP test with all five criteria.

⚡ Core Intuition

CRAAP = 5-question screen for any source: is it Current, Relevant, Authoritative, Accurate, Purposeful?

📖 Concept Explained
  • Developed by Sarah Blakeslee, Meriam Library, CSU Chico (2004).
  • Quick checklist before citing any source — book, article, website, post.
  • Each criterion = one filter; weak in 2+ ⇒ reject source.
  • Fast version of broader information literacy.
🔗 Connects to credible sources bias detection info literacy evidence-based argument
📐 Exam Answer Structure
  • Definition + who coined it (1 line).
  • Draw pentagon diagram with 5 labels.
  • Table: criterion → key questions to ask.
  • Applied example (Wikipedia article).
  • Conclusion: why CRAAP saves time.
LetterCriterionKey question
CCurrencyWhen was it published / last updated? Is it still valid?
RRelevanceDoes it match my topic and audience level?
AAuthorityWho wrote it? Credentials? Reputable publisher?
AAccuracyAre facts cited and verifiable? Any errors?
PPurposeInform, sell, persuade, entertain? Hidden agenda?
💡 Worked Example — Evaluate a Wikipedia article on "Climate Change"
  • C: Last edited yesterday ⇒ Current ✓
  • R: Matches "climate change basics" essay ⇒ Relevant ✓
  • A: Anonymous editors ⇒ Authority ✗ (cite original cited papers instead).
  • A: 300+ footnoted citations to peer-reviewed journals ⇒ Accurate ✓
  • P: Educational, no ads ⇒ Purpose ✓
  • Verdict: usable as starting point; cite original sources for academic work.
🧠 Mnemonic — "Crap sources are CRAAP-tested" → C urrency, R elevance, A uthority, A ccuracy, P urpose.
✅ Model Exam Answer (10–15 marks)
  1. Definition: CRAAP is a 5-criterion checklist developed by Sarah Blakeslee (2004, CSU Chico) for evaluating any information source.
  2. Diagram: pentagon with C, R, A, A, P around central "Source".
  3. Five criteria: Currency (date), Relevance (fit to topic), Authority (author credentials), Accuracy (verifiable facts), Purpose (intent / bias).
  4. Applied example: Wikipedia article passes 4/5; fails Authority due to anonymous editors.
  5. Significance: prevents misinformation, supports academic integrity, sharpens information literacy.

📚 Primary: Content Analysis and Articulation (notes) · Supplemented: Blakeslee S. (2004), CSU Chico Meriam Library.

Q2 How do you identify a credible source? Factors + examples.

⚡ Core Intuition

Q1 The STAR Method — Situation, Task, Action, Result

⚡ Core Intuition

  • Give 1 full worked answer (any behavioural Q).

Definition

STAR is a four-step storytelling framework used to structure answers to behavioural interview questions.

💡 Worked Example 1 — "Tell me about a time you handled a tight deadline"
  • S: Final-year DBMS project, 3 days before submission, my team member fell sick.
  • T: I had to finish his ER diagram + 2 stored procedures alone, without missing my own modules.
  • A: Re-planned the timeline into 6-hour blocks, used Lucidchart for ER, paired with a senior over WhatsApp for SQL review, slept 4 hrs/day for 2 days.
  • R: Submitted on time; project scored 92/100 — highest in the section; learnt to keep a backup teammate from day one.

  • R: Event ran on schedule, zero drop-outs, sponsor renewed for next year; principal cited it in convocation speech.
Tip — Karpathy Rule

Always answer in S→T→A→R order. Weight Action 40% — that's where YOU prove you did the work.

Q1 Explain the 5 conflict resolution strategies — avoid, compromise, accommodate, compete, collaborate.

⚡ Core Intuition

Conflict strategies sit on 2 axes — how much you push your needs (assertiveness) vs how much you help others (cooperativeness).

  • Compete = high me, low them (win–lose)
  • Accommodate = low me, high them (lose–win)
  • Collaborate = high both (win–win)
  • Avoid = low both (lose–lose)
  • Compromise = medium both (partial win)
📖 Concept Explained

Model proposed by Thomas & Kilmann (TKI grid).

  • No single strategy is "best" — fit depends on stakes & relationship.
  • Collaborate gives best outcome but costs time.
  • Compete protects rights but damages bonds.
  • Avoid is valid only when issue is trivial.
🔗 Connects to 3 P's Negotiation Active listening Leadership
📐 Exam Answer Structure
  • Define conflict (1 line).
  • Draw 2×2 TKI grid (mandatory).
  • 5 strategies — definition + when-to-use + example each.
  • Compare table: pros/cons.
  • Conclude: choice depends on stakes & relationship.
💡 Example (each strategy)
  • Compete: Manager enforces safety rule despite worker objection.
  • Accommodate: Yield seat choice to a colleague for goodwill.
  • Avoid: Ignore a trivial argument over canteen menu.
  • Compromise: Two teams split a shared budget 50–50.
  • Collaborate: Engineering & sales co-design a product that meets both targets.

AvoidTrivial issue, need cool-offSaves energyIssue resurfaces
CompeteEmergency, ethics, rightsFast, decisiveDamages bonds
AccommodateOther's need urgentPreserves harmonyBuilds resentment
CompromiseEqual power, time shortQuick fairnessBoth partly unhappy
CollaborateBig stakes, long relationBest outcome, trust ↑Slow, needs effort

Q2 Explain the 3 P's of conflict resolution — Problem, People & Process.

⚡ Core Intuition

Every conflict has three layers — what (Problem), who (People), how (Process). Fixing only one fails.

  • Problem — the issue itself.
  • People — emotions, perspectives, needs.
  • Process — method to talk it through.
📖 Concept Explained
  • Problem: Identify root cause; separate symptom from cause.
  • People (Perspective): Understand each party's interest & emotion; "soft on people, hard on problem".
  • Process: Structured method — communication, negotiation, mediation, compromise.
  • Missing People = solution rejected. Missing Process = chaos. Missing Problem clarity = wasted effort.
🔗 Connects to 5 strategies Active listening Mediation

  • People: One feels overworked, other fears boss's review.

Q7 Explain Tuckman's stages of team development.

⚡ Core Intuition

Every team passes through 5 predictable stages — Forming → Storming → Norming → Performing → Adjourning.

  • Conflict (Storming) is normal — not a failure.

  • Performing — high output, trust, autonomy.
  • Adjourning — task done, team dissolves.

Red Flags — Common Penalties

Read this 5 minutes before entering the hall. Each costs 1–3 marks.

  • Naming wrong hat colour: Blue = process control, NOT calm/water. Yellow = optimism, NOT caution. Mix-ups lose 1 mark per wrong hat.
  • Forgetting Edward de Bono's name: Every long answer on hats must credit "Edward de Bono (1985)" in the definition line.
  • Confusing inductive and deductive direction: Inductive = specific → general (probable). Deductive = general → specific (certain). Swapping = full mark loss.
  • STAR with no example: Listing S/T/A/R without a concrete scenario = half marks max. Examiner expects a project / team / deadline story.
  • STAR with too much "Situation": Action should be 60% of the answer. Long context = penalised.
  • Missing one CRAAP letter: All 5 (C-R-A-A-P) must be present in any source-credibility answer. One missing = 1 mark off.
  • Calling CRAAP "criteria for writing": CRAAP is for source EVALUATION, not for writing. Wrong scope.
  • Conflict strategies without Thomas-Kilmann attribution: The 5 strategies have a named author — cite "Thomas & Kilmann (1974)".
  • 3 P's wrong expansion: Problem · People · Process — NOT "Plan, Prepare, Proceed". The paper specifies it.
  • Listing 4 Tuckman stages: There are FIVE (Forming, Storming, Norming, Performing, Adjourning). Skipping Adjourning loses a mark.
  • Bias eliminated with "be neutral": Vague. Examiner wants: peer review, triangulate sources, blind data collection, awareness training.
  • Rhetorical situation with only 2 elements: All 4 (Purpose · Audience · Topic · Context) must appear. Listing only 2 = half marks.
  • Paragraph answers instead of bullets: Examiners read fast — paragraphs hide structure. Use numbered points + 1-line definitions.
  • Skipping the definition line: Every long answer must open with a 1-sentence textbook-style definition. Missing = 1 mark off.
  • No diagram on draw-trigger questions: "Explain Six Hats / STAR / Conflict strategies" → diagram = half the marks. No diagram = capped at 50%.
  • Observation/inference confusion: "I see dark clouds" = observation. "It will rain" = inference. Don't merge into one sentence.
  • Mock interview = real interview: Mock is a simulation for practice with feedback. Don't describe it as actual hiring.
